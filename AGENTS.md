## OVERVIEW

Expo SDK 56 + React Native 0.81.5 + React 19.1 monorepo for a BSSM school indoor/outdoor map app. MapLibre renders the campus; BLE WCL (iOS-only, foreground) and Android WiFi RTT fuse with dead reckoning to localize indoors; a custom pathfinder walks the BSSM floor data for routing.

## CONVENTIONS (DEVIATIONS FROM DEFAULT RN/EXPO)

- **TypeScript strict** + `allowArbitraryExtensions`; tests excluded from `tsc` compile.
- **No ESLint, Prettier, EditorConfig** — relies on defaults. Don't propose adding them without asking.
- **No path aliases** — all imports are relative. tsconfig just extends `expo/tsconfig.base`.
- **Metro** treats `.mbtiles` as assets, `.geojson` as source, via `metro-geojson-transformer.js`.
- **Jest** uses `jest-expo` preset + the same `.geojson` transform.
- **Babel** adds `react-native-worklets/plugin` (required for reanimated v4).
- **pnpm** with `node-linker=hoisted` (set in `.npmrc`) — required for Expo SDK 56 monorepo.
- **No CI workflows** — no `.github/workflows/*`, no `Makefile`, no `bitrise.yml`. Deploy via `eas.json` profiles.

## ANTI-PATTERNS (THIS PROJECT)

1. **NEVER modify the baseline RSSI weight formula** in `src/services/location/bleWeightedCentroid.ts` — it is the calibration baseline for all BLE WCL positioning.
2. **NEVER fold `accessibilityPenalty` into `timeSeconds` or `effortMetersEquivalent`** — it is a preference-only channel for `elevator_priority` mode (see `src/services/routing/pathfinder.ts:80-99` and `src/types/routing.ts:55-70`).
3. **NEVER use `setUserCoordinates`** — it is `@deprecated`; use `setGpsCoordinates` instead (`src/store/mapStore.ts:60`).
4. **NEVER assume iOS Core Location is accurate indoors** — requires calibration (`src/services/calibration/iosCalibration.ts`) or anchors.
5. **NEVER treat BLE WCL as background or Android-capable** — it is **foreground-only, iOS-only**. The Android path uses WiFi RTT instead.
6. **NEVER conflate the two coordinate pipelines**:
   - Legacy RTT/SVG operates in **map-percent** (`0..1` x/y).
   - BLE WCL operates in **WGS84** `[longitude, latitude]`, sourced from EPSG:5183 via `proj4`.
7. **`.omo/`, `.sisyphus/`, agent state dirs are gitignored** — never commit inside them.

## UNIQUE STYLES

- **Two parallel coordinate pipelines** (see `README.md` "Coordinate Systems" section). The README's "Verify a coordinate's source" recipe is the authoritative way to disambiguate.
- **BLE WCL = WGS84 only**; legacy RTT/SVG = map-percent only. Conversion only via calibration/anchors.
- **Routing is a multi-option engine** (Yen's algorithm) with three profiles: `fastest` | `shortest` | `easiest`. Accessibility mode is orthogonal (`normal` | `elevator_priority`).
- **Cross-screen UI signals** use monotonic tick counters (`showAttributionTick`, `minimizeSheetsTick`) in `mapStore` — not boolean UI state.
- **Stores are zustand 5** — no Redux/Recoil/MobX.
- **Path alias-less imports** — every `import` uses `../` relative paths.
- **Custom Expo config plugins** mutate the prebuild native config; all three are committed in `plugins/`.

## NOTES

- **No Android host directory is checked in.** `android/` is in `.gitignore`; the Android host is generated by `expo prebuild` from `app.config.js`. Custom native code lives under `modules/`.
- **The project uses Expo's "new architecture"** (`newArchEnabled: true` in `app.json`). Local modules must use the new TurboModule/Fabric API.
- **Floor data is MIT-licensed from `obtuse-triangle/BSSM-Map`** — preserve attribution in derived constants.
- **iOS BLE delivery can lag 5s to 1+ min** even with 1s AP advertising interval (per `README.md` "iOS limitation" + `docs/ble-wcl-mvp.md`). Do not treat stale-but-fresh data as a bug.
- **`docs/` runbooks are authoritative for QA flows**: `ble-wcl-mvp.md`, `ble-motion-fusion.md`, `indoor-routing-qa.md`, `android-ble-field-qa.md`, `ap-data-collection-guide.md`, `ble-fusion-field-qa.md`, `data-flow.md`.
- **`.omc/notepad.md` is always-loaded KB memory** — keep it under 500 chars; treat as critical-discovery only.
- **Routing GeoJSON is committed**, not generated at build time. Run `scripts/validate-routing-data.js` after any data change.
- **Map data is the source of truth**, not React state — MapLibre layer/source updates bypass React for perf.
