## QGIS-Editable Files

### `src/data/routing-walkable-areas.geojson`

56 walkable polygon features representing the walkable interior space on each floor.

| Floor | Features | Description |
|-------|----------|-------------|
| 1 | 17 | Ground floor walkable areas |
| 2 | 18 | Second floor walkable areas |
| 3 | 2 | Third floor walkable areas |
| 4 | 19 | Fourth floor walkable areas |

**Required properties per feature:**
- `level` (number): Floor level (1-4)
- `areaSquareMeters` (number): Planar area in EPSG:5183
- `sourceFeatureIds` (string[]): Original campus feature IDs that defined the outline/obstacles

**Geometry:** WGS84 `Polygon` coordinates (standard GeoJSON).

### `src/data/routing-connectors.geojson`

26 connector point features linking floors via stairs and elevators.

| Type | Count | Accessibility |
|------|-------|---------------|
| Stair | 22 | Penalty > 0 (5) |
| Elevator | 4 | Penalty = 0 |

**Required properties per feature:**
- `connectorType`: `"stair"` or `"elevator"`
- `connectsLevels`: `[fromLevel, toLevel]` tuple (adjacent for stairs, full-range for elevators)
- `traversalTimeSeconds`: Estimated time to traverse (stairs: 15s, elevators: 30s)
- `accessibilityPenalty`: 0 for elevators, >0 for stairs
- `sourceFeatureIds` (string[]): Original campus feature IDs for this connector
- `confidence`: `"auto"` (generated by script) or `"manual"` (user-refined)

**Geometry:** WGS84 `Point` coordinates.

## Regeneration Commands

These build-time scripts read source data and regenerate the routing GeoJSON files:

```bash
# Regenerate walkable areas (subtract obstacles from school outline)
node scripts/generate-walkable-areas.js

# Regenerate stair/elevator connectors
node scripts/generate-routing-connectors.js

# Validate output
node scripts/validate-routing-data.js
```

**Source data:**
- `src/data/school-outline.json` — Building exterior footprint (18 MultiPolygon features)
- `src/data/campus-wgs84.json` — Campus features (411 features: rooms, stairs, elevators, facilities)

**Obstacle categories** (subtracted from outline to produce walkable areas):
`classroom`, `room`, `restroom`, `facility`, `structural`, `unknown`

**Connector candidates:**
`stair` (32 features) and `elevator` (13 features), grouped by 15m proximity threshold.

### Determinism

Regenerated data should be identical unless:
1. Source data (`school-outline.json` or `campus-wgs84.json`) changed
2. QGIS refinements were applied to the committed GeoJSON files
3. The generation script logic was modified

The graph builder is fully deterministic — same input always produces same node IDs (`f{level}-r{ringIdx}-{ptIdx}`, `f{level}-g{x}-{y}`, `conn-{type}-{index}-{level}`).

## Data Maintenance Notes

### After QGIS Editing

1. Open `routing-walkable-areas.geojson` or `routing-connectors.geojson` in QGIS
2. Edit polygon shapes, connector positions, or property values
3. Save back to GeoJSON format (WGS84 coordinates)
4. Run `node scripts/validate-routing-data.js` to verify structure
5. Run `npm test -- --runInBand` to verify graph integrity

### Runtime Graph Rebuild

The routing graph is rebuilt at runtime from the committed GeoJSON files via `buildRoutingGraph()`:
- **Nodes:** 4,543 (polygon ring vertices + grid interior samples + connector nodes)
- **Edges:** 29,636 (walk edges + cross-floor connector edges)
- **Build time:** ~760ms (cached after first call)
- **Grid spacing:** 2.0m (configurable)
- **KNN edges:** K=6 nearest neighbors within 2× spacing, midpoint containment verified

### Coordinate System

- **Routing math:** EPSG:5183 (Korean TM Central Belt) — planar meters
- **Source data:** WGS84 (standard GeoJSON)
- **MapLibre rendering:** WGS84
- **Transforms:** `transformWgs84ToEpsg5183()` / `transformEpsg5183ToWgs84()` in `src/utils/coordinateTransform.ts`

## Architecture Summary

### Routing Services (`src/services/routing/`)

| File | Purpose |
|------|---------|
| `graphBuilder.ts` | Builds routing graph from GeoJSON data (grid sampling, KNN edges, connector linkage) |
| `graphValidator.ts` | Validates graph integrity (edge refs, weights, penalties, orphans) |
| `pathfinder.ts` | Dijkstra shortest-path with MinHeap, accessibility weighting |
| `coordinateSnap.ts` | Snaps WGS84 coordinates to nearest graph node (20m threshold) |
| `routeComputer.ts` | Orchestrates origin/dest resolution, temp node injection, pathfinding |
| `routeGeoJson.ts` | Converts RouteResult → WGS84 GeoJSON FeatureCollection |
| `geojson-transform.js` | Jest transformer for `.geojson` imports |

### State Management

- `src/store/routeStore.ts` — Zustand store with route lifecycle (origin, destination, result, accessibility mode)
- Injectable route computer: `setRouteComputer(fn)` for testing

### UI Components

| File | Purpose |
|------|---------|
| `src/components/map/RoutePathLayer.tsx` | MapLibre overlay: active (#2979FF) + dimmed (#999) route layers |
| `src/components/map/routeLayerData.ts` | Pure function: classifies features by current floor |
| `src/screens/PlaceDetailSheetScreen.tsx` | 길찾기 button + 출발지로 설정 action |
| `src/screens/MapSheetScreen.tsx` | Route card with distance, floor info, elevator toggle, clear |

### Build-Time Scripts (`scripts/`)

| File | Purpose |
|------|---------|
| `generate-walkable-areas.js` | Projects outline + obstacles to EPSG:5183, subtracts via polygon-clipping |
| `generate-routing-connectors.js` | Groups stairs/elevators by proximity, generates connector features |
| `validate-routing-data.js` | Standalone validator for committed GeoJSON files |

## Scope Guards

These files must NOT be modified by routing work:

- `src/constants/bssmFloorMap.ts`
- `src/components/map/NativeFloorMap.tsx`

Routing must not add outdoor routing, turn-by-turn navigation, live rerouting, or a third route formSheet.
